## Project: MonkeySwim
**Spec (`specs/MonkeySwim.md`):**
- Android-native game, gameplay nearly identical to Pac-Man.
- Multiple levels; level 1 essentially identical to original Pac-Man maze.
- Three gateways: left, right (wrap), and a bottom gateway that is **locked** until last pellet eaten — entering it transitions to next level.
- Player is a monkey; enemies are piranhas (replacing ghosts).
- Water-themed: canals + islands instead of arcade maze walls.
- Sprite animations for monkey + piranhas swimming up/down/left/right with transitional frames.
- 3 lives. On game over, splash screen offers "Restart" or "Watch Ad (Google AdMob) for +3 lives".

## Decisions
- **Framework:** Kotlin + Android SDK with `SurfaceView`/custom game thread (textbook Android-native for 2D games). No LibGDX/Unity.
- **Sprites v1:** Programmatically drawn (Canvas) placeholder sprites with 4 directions × 2-3 frames. Designed so a sprite sheet can be swapped in later.
- **Piranha AI v1:** Simplified Pac-Man ghost personalities (chase/scatter/random) — refined later.
- **AdMob:** SDK wired with Google's official **test** rewarded-ad unit ID; clear TODO to swap in real publisher IDs. App ID placeholder in manifest.
- **Levels v1:** Single hand-laid 28x36 grid mirroring original Pac-Man. Levels 2+ reuse layout with faster piranhas until more layouts are added.
- **Build env:** Project will be opened/built in Android Studio. Gradle wrapper jar/scripts not generated by me — Android Studio adds them on first sync.
